# Sample Project

This sample project exists to test Truenex Memory quickstart flows.

It documents a small local-first coding agent setup. The important decisions are
stored in `docs/architecture.md` and `docs/decisions.md` so indexing can return
source paths, headings, and scores.

Useful questions:

- Which database stores local metadata?
- Which vector database is planned?
- Which MCP transport is used?
- Are code and memory uploaded automatically?
